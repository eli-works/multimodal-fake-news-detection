## 2. Related Technologies

### 2.1 文本表示学习技术

- 文本表示学习是多模态假新闻检测的基础，其目标是将离散词序列映射为具有语义信息的连续向量表示
- 静态词向量能够提供基础语义信息，但难以充分建模上下文变化
- 循环神经网络，尤其是 LSTM 和 BiLSTM，擅长捕捉序列中的局部上下文关系与时序依赖
- Transformer 通过自注意力机制建模长距离依赖，能够更好提取全局语义信息
- 在本研究中，BiLSTM 与 Transformer 被组合为双路径文本建模模块，用于同时保留局部与全局语义特征

### 2.2 图像特征提取技术

- 图像模态为假新闻检测提供视觉证据，可用于识别配图内容、场景语义及潜在的误导性线索
- 卷积神经网络是图像表征学习的主流方法，能够从局部纹理逐步提取高层语义特征
- EfficientNet 系列通过复合缩放策略在精度与计算效率之间取得较好平衡
- EfficientNet-B0 参数规模适中，适合作为多模态任务中的视觉编码器
- 本研究采用 ImageNet 预训练的 EfficientNet-B0 提取图像特征，并通过阶段冻结策略保留通用视觉知识

### 2.3 跨模态表示对齐技术

- 文本特征与图像特征通常分布在不同语义空间，直接融合容易受到模态差异影响
- 跨模态表示对齐的目标是在共享空间中缩小不同模态之间的语义距离
- 常见方法包括共享投影、对比学习、对抗对齐以及残差映射等
- 残差式映射结构能够在保留原始特征的基础上引入对齐信息，减少信息损失
- 本研究设计共享残差桥模块，将文本与图像特征映射到更一致的语义空间，为后续交互奠定基础

### 2.4 跨模态注意力融合技术

- 多模态任务的核心在于建模不同模态之间的互补关系与一致性关系
- 简单拼接虽然实现方便，但难以显式捕捉图文之间的细粒度关联
- 注意力机制能够根据一个模态的信息动态选择另一个模态中的关键线索
- 双向跨模态注意力进一步支持文本到图像、图像到文本的双向信息流动，更适合发现图文不一致现象
- 本研究使用双向跨模态注意力模块，对共享语义空间中的文本与图像特征进行深层交互融合

## 3. Method

### 3.1 模型总览

- 任务目标：输入文本 `T` 与图像 `I`，预测新闻真假
- 整体流程：

```text
输入
  -> 数据预处理
  -> 文本分支 (BiLSTM + Transformer) || 图像分支 (EfficientNet-B0)
  -> 共享语义桥 (Shared Bridge)
  -> 双向跨模态注意力 (BiCrossAttn)
  -> 分类头 (MLP)
  -> 真/假输出
```

- 图示建议：Figure 1. Overall architecture of the proposed model

### 3.2 文本特征提取模块

#### 3.2.1 设计动机

- BiLSTM 擅长建模局部上下文与顺序依赖
- Transformer 擅长建模全局语义与长距离依赖
- 双路径互补：先利用 BiLSTM 编码局部信息，再通过 Transformer 建模全局关系

#### 3.2.2 流程

```text
T
  -> Embedding (L x 192)
  -> BiLSTM (hidden = 128 per direction, output = 256)
  -> Linear (256 -> 192) + Positional Encoding
  -> Transformer Encoder (4 heads, 1 layer)
  -> Masked Mean Pooling
  -> f_T (192)
```

#### 3.2.3 关键公式

| 编号 | 公式 | 说明 |
| --- | --- | --- |
| (1) | `E_T = Embedding(T), E_T in R^(L x 192)` | 词嵌入 |
| (2) | `H_lstm = BiLSTM(E_T), H_lstm in R^(L x 256)` | 双向 LSTM |
| (3) | `H_proj = W_proj * H_lstm + b_proj, H_proj in R^(L x 192)` | 投影 |
| (4) | `H_trans = TransformerEncoder(H_proj + PE), H_trans in R^(L x 192)` | 全局语义 |
| (5) | `f_T = (1 / M) * sum(H_trans * M), f_T in R^192` | 掩码池化 |

### 3.3 图像特征提取模块

#### 3.3.1 设计动机

- EfficientNet-B0 轻量高效，预训练特征表现稳定
- 冻结前若干阶段，保留通用视觉表征，减少过拟合风险

#### 3.3.2 流程

```text
I
  -> EfficientNet-B0 (frozen stages 0-2)
  -> h_I (1280)
  -> Linear (1280 -> 192) + LN + GELU + Dropout
  -> f_I (192)
```

#### 3.3.3 关键公式

| 编号 | 公式 | 说明 |
| --- | --- | --- |
| (6) | `h_I = EfficientNet(I), h_I in R^1280` | 视觉特征 |
| (7) | `f_I = Dropout(GELU(LN(W_img * h_I + b_img))), f_I in R^192` | 图像投影 |

### 3.4 跨模态特征融合模块

#### 3.4.1 共享残差桥

- 动机：缓解文本语义空间与图像视觉空间的分布差异
- 结构：两层共享全连接层 + Dropout
- 作用：完成模态对齐并增强共有语义表达

| 编号 | 公式 |
| --- | --- |
| (8) | `Res_x = Dropout(W2 * tanh(W1 * x + b1) + b2)` |
| (9) | `t_share = tanh(Dropout(t) + alpha * Res_t)` |
| (10) | `v_share = tanh(Dropout(v) + alpha * Res_v)` |

#### 3.4.2 双向跨模态注意力

- 动机：捕捉图文语义不一致与互补线索
- 结构：文本到图像、图像到文本双向交互
- 作用：提升细粒度跨模态信息融合效果

| 编号 | 公式 |
| --- | --- |
| (11) | `H_t->v = Softmax(Q_t * K_v^T / sqrt(d_h)) * V_v` |
| (12) | `H_v->t = Softmax(Q_v * K_t^T / sqrt(d_h)) * V_t` |

#### 3.4.3 融合拼接

- 文本侧：掩码池化得到 `g_T`
- 图像侧：均值池化得到 `g_I`
- 最终拼接：`Z = g_T concat g_I`, `Z in R^384`

| 编号 | 公式 |
| --- | --- |
| (13) | `H_image = Reshape(W_proj * v_share + v_share) in R^(4 x 192)` |
| (14) | `Z = g_T concat g_I, Z in R^384` |

### 3.5 预测与损失

#### 3.5.1 分类头

- `Z (384) -> Linear (384 -> 128) + LN + GELU + Dropout -> h_cls`
- `h_cls -> Linear (128 -> 2) + Softmax -> y`

#### 3.5.2 损失函数

- 使用二分类交叉熵损失

| 编号 | 公式 |
| --- | --- |
| (15) | `h_cls = GELU(LN(W_c1 * Z + b_c1))` |
| (16) | `y = Softmax(W_c2 * h_cls + b_c2)` |
| (17) | `L_CE = -(1 / N) * sum(sum(y_ic * log(y_hat_ic)))` |

#### 3.5.3 优化方式

- 优化器：AdamW
- 训练方式：端到端联合训练

### 4.6 消融实验

#### 4.6.1 实验设计

| 变体 | 说明 |
| --- | --- |
| Ours | 完整模型 |
| LSTM | 去掉 Transformer 文本路径 |
| TRM | 去掉 BiLSTM 文本路径 |
| -BCA | 去掉 BiCrossAttn |
| -SB | 去掉 Shared Bridge |

#### 4.6.2 关键发现

| 变体 | 影响最大数据集 | 退化幅度 | 原因分析 |
| --- | --- | --- | --- |
| -SB | Weibo | F1 -2.83% | 共享桥对语义对齐关键 |
| -SB | Gossip | 真新闻召回 -25.57% | 少数类识别依赖共享桥 |
| -BCA | CFND | P -5.23% / R +4.35% / F1 -0.76% | 缺少细粒度交互导致误报增加 |
| LSTM | Weibo | F1 -8.23% | 单路径文本建模不稳定 |
| TRM | Weibo | F1 -4.41% | 单路径文本建模不稳定 |

#### 4.6.3 总体结论

- 双路径文本建模、共享桥与跨模态注意力共同支撑模型表现
- 单路径文本变体退化最明显

### 4.7 参数分析

#### 4.7.1 EfficientNet 冻结层数

- 分析 `freeze_efficientnet_stages` 从 0 到 9 的影响

| 数据集 | 准确率波动 | 原因 |
| --- | --- | --- |
| Gossip | 0.85% | 文本线索更强，图像样式相对单一 |
| CFND | 1.76% | 图像语义与预训练分布较匹配 |
| Weibo | 4.36% | 图像来源复杂，更依赖任务适配 |

- 结论：
  - CFND 更适合较高冻结层
  - Gossip 与 Weibo 更适合较低冻结层
